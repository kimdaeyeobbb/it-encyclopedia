# 자바스크립트 딥다이브 개념 발표

## 03. 데이터 타입

### 3.1 데이터 타입의 종류

|     구분      |                                     데이터타입                                     |
| :-----------: | :--------------------------------------------------------------------------------: |
|               |                                 숫자(Number) 타입                                  |
|               |                                문자열(String) 타입                                 |
|               |                                불리언(Boolean) 타입                                |
| **원시 타입** |                                   undefined 타입                                   |
|               |                                     null 타입                                      |
|               |                                 심벌(Symbol) 타입                                  |
|               |                                    BigInt 타입                                     |
| **객체 타입** | &nbsp;&nbsp;&nbsp; 객체(Object), 함수(Function), 배열(Array) 등 &nbsp;&nbsp;&nbsp; |
|               |                                                                                    |

<br>

- 숫자 타입
  - 정수, 실수 구분 없이 하나의 숫자 타입만 존재
  - 상징적인 3가지 값
    - `Infinity` : 양의 무한대
    - `-Infinity` : 음의 무한대
    - `NaN` (Not a Number) : 산술 연산 불가

<br>

- 문자열 타입
  - 텍스트 데이터
  - 표현: `''`(작은 따옴표), `""`(큰 따옴표), ` `` `(템플릿 리터럴)
  - 템플릿 리터럴
    - ES6부터 도입된 새로운 문자열 표기법
    - 줄바꿈과 모든 공백이 그대로 적용됨
    - 표현식을 `${}`으로 감싸면 표현식이 문자열 타입으로 변환됨

<br>

- 불리언 타입
  - 논리적 `true`(참)과 `false`(거짓)을 나타냄

<br>

- undefiend 타입
  - `var` 키워드로 선언한 변수를 암묵적으로 초기화할 때 사용하는 값
  - 개발자가 사용하는 것은 지양

<br>

- null 타입
  - 변수에 값이 없다는 것을 의도적으로 명시할 때 사용
  - 함수가 유효한 값을 반환할 수 없는 경우 명시적으로 `null`을 반환하기도 함

<br>

- 심벌 타입
  - ES6에 추가된 타입
  - 다른 값과 중복되지 않는 유일무이한 값
  - Private 객체의 프로퍼티 키로 사용됨
  - `Symbol()` 함수를 호출해 생성

<br>

- BigInt 타입

  - ES2020에 추가된 숫자 원시 값
  - `MAX_SAFE_INTEGER` (숫자 타입의 안전 한계)를 넘어서는 정수를 안전하게 저장하고 연산 가능
  - 정수 끝에 `n`을 추가하거나 `BigInt()` 함수를 호출해 생성
  - 숫자 타입처럼 연산자를 사용 가능하지만 숫자 타입과 혼합해 연산할 경우, TypeError가 발생

    ```js
    const bigint = 1234567890123456789012345678901234567890n;

    const sameBigint = BigInt("1234567890123456789012345678901234567890");

    const bigintFromNumber = BigInt(10);
    // 10n과 동일
    ```

<br>

- 객체 타입
  - 자바스크립트는 객체 기반의 언어
    → 원시 타입 외의 값은 모두 객체 타입
  - 자세한 내용은 <07 객체 리터럴>과 <08 원시값과 객체 비교>에서 다룰 예정

<br>
<br>

### 3.2 데이터 타입이 필요한 이유

- 값을 저장할 때 확보해야 하는 **메모리 공간의 크기**를 결정하기 위해
- 값을 참조할 때 한 번에 읽어 들여야 할 **메모리 공간의 크기**를 결정하기 위해
- 메모리에서 읽어 들인 **2진수를 어떻게 해석**할지 결정하기 위해 _→ 0100 0001 은 숫자 65이자 문자열 ‘A’_

### 3.3 동적 타이핑

#### 3.3.1 동적 타입 언어와 정적 타입 언어

- 정적 타입 언어: 변수를 선언할 때 할당할 값의 데이터 타입을 사전에 선언
  - 변수의 타입을 **변경 불가**
  - 컴파일 시점에 **타입 체크**(선언한 데이터 타입에 맞는 값을 할당했는지 검사)를 수행
  - 언어: C, C++, 자바, 코틀린 등
- 동적 타입 언어: 타입을 선언하지 않고 어떤 값이든 자유롭게 할당
  - 선언이 아닌 **할당에 의해 타입이 결정(타입 추론)**됨
  - 재할당에 의해 변수의 타입은 언제든지 동적으로 변할 수 있음 → **동적 타이핑**
  - 언어: **자바스크립트**, 파이썬, 루비 등

<br>

#### 3.3.2 동적 타입 언어와 변수

- 단점
  - 변화하는 변수 값을 추적하기 어려움
  - 자바스크립트 엔진에 의해 타입이 자동으로 변환되기도 함
    <br>
    → 유연성은 높으나 신뢰성이 떨어짐
- **변수를 사용할 때 주의 사항**
  - 변수는 필요한 경우에 한해 제한적으로 사용
  - 변수의 유효 범위(스코프)는 최대한 좁게 만들어 부작용 억제
  - 전역 변수는 최대한 사용 지양
  - 변수보다 상수(const)를 사용해 값의 변경을 억제
  - 변수 이름은 목적과 의미를 파악할 수 있도록 네이밍

<br>
